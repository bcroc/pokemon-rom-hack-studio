# PHS-T128 App Icon

## Summary

`PHS-T128` adds an original pixel-editor macOS app icon to the PokemonHackStudio app target. The tracked `Assets.xcassets/AppIcon.appiconset` contains the full macOS icon slot set derived from a generated square master image, and `project.yml` now declares `ASSETCATALOG_COMPILER_APPICON_NAME: AppIcon` as the durable XcodeGen source of truth.

The row changes only app bundle resources and XcodeGen configuration. It adds no Swift API, CLI command, parser behavior, schema shape, source-write path, patch/export authority, ROM write, reference edit, or mutation-plan behavior.

## Proof

- Built-in image generation created the original pixel-editor master image under the default Codex generated-image directory; the project keeps the derived 1024px master as `icon_512x512@2x.png` plus the smaller macOS slots.
- `sips -g pixelWidth -g pixelHeight PokemonHackStudio/Sources/PokemonHackStudio/Assets.xcassets/AppIcon.appiconset/*.png` confirmed the expected 16, 32, 64, 128, 256, 512, and 1024 pixel outputs.
- `plutil -lint PokemonHackStudio/Sources/PokemonHackStudio/Assets.xcassets/Contents.json PokemonHackStudio/Sources/PokemonHackStudio/Assets.xcassets/AppIcon.appiconset/Contents.json` was attempted, but this machine's Swift-mode `plutil` rejected raw JSON with `Unexpected character { at line 1`; `python3 -m json.tool` parsed both files successfully.
- `cd PokemonHackStudio && xcodegen generate` passed and regenerated the ignored project with `Assets.xcassets` in Resources and `ASSETCATALOG_COMPILER_APPICON_NAME = AppIcon`.
- `make verify` passed; it regenerated plists/project, built/signed the macOS app, ran the bundle-assets phase, registered the app, and completed with `BUILD SUCCEEDED`.
- Built app confirmation: `DerivedData/PokemonHackStudio/Build/Products/Debug/PokemonHackStudio.app/Contents/Info.plist` contains `CFBundleIconFile => AppIcon` and `CFBundleIconName => AppIcon`; the bundle contains `Contents/Resources/AppIcon.icns` and `Assets.car`, and `xcrun assetutil --info .../Assets.car` lists `AppIcon` renditions for all tracked icon slots.
